Fit/Gap for AI Agent (POTENTIAL FUTURE STATE)

Fit

  • Must respond conversationally to users' questions.
  • Must have a source of knowledge that is separate from Support KB articles (e.g., “Workflow Helpers”).
  • Must have the ability to provide the AI with custom instructions so it can tailor its messages to a specific audience.
  • Should show typing indicators when “thinking” about an answer.

Gap

  • Should greet the customer when they start a chat with a first outbound message (AI Agent only responds with the customer once the first message has been sent).
  • Should allow us to call functions that can be used in Workflows to kick off process automations (e.g., when a user intends to talk to sales, schedule a demo, or talk to support, we use a workflow to send them messages, notify the necessary stakeholders in Slack, and route accordingly).
  • Should show the "bot" word beneath the AI agent messages per legal team’s recommendation.
  • Should keep a conversation open without marking it done arbitrarily, preventing a user from spamming us by creating new conversations.
  • Could use proactive messaging to pop up a message bubble that greets a customer when they visit the site.
  • Could allow non-public data sources to be uploaded to ensure data stays private.
  • Could Use clearbit/apollo for reverse ip lookup
